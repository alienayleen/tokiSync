# 🛠️ 신규 사이트 파서 개발을 위한 DOM 구조 추출 가이드

새로운 사이트의 파서를 개발하기 위해서는 크게 **1) 작품 메인 페이지(에피소드 목록 페이지)**와 **2) 뷰어 페이지(실제 웹툰/소설을 보는 페이지)** 두 곳의 HTML 구조가 필요합니다.

브라우저의 개발자 도구(F12)를 열어 필요한 요소를 찾은 뒤, 우클릭하여 `Copy -> Copy element (또는 outerHTML)`를 통해 아래 영역들의 HTML 코드를 복사해서 제공해 주세요.

---

## 1. 작품 메인 페이지 (에피소드 목록 페이지)

### ① 에피소드 목록 영역 (List Items)
- **필요한 것:** 에피소드 목록을 감싸고 있는 전체 컨테이너 요소와, 그 안에 나열된 **개별 에피소드 항목 1~2개**의 HTML 구조
- **추출 목적:** 에피소드의 번호(몇 화인지), 제목, 해당 에피소드로 이동하는 링크(URL)를 추출하기 위함 (`getListItems`, `parseListItem` 구현)

### ② 작품 메타데이터 및 썸네일 영역 (Metadata & Thumbnail)
- **필요한 것:** 작품의 제목, 썸네일 이미지(`<img>`), 작가, 연재 상태, 줄거리가 포함된 HTML 영역 또는 문서 헤더의 `<meta>` 태그들
- **추출 목적:** 올바른 다운로드 폴더명을 생성하고 썸네일을 추출하기 위함 (`getSeriesTitle`, `getThumbnailUrl`, `getSeriesMetadata` 구현)

---

## 2. 뷰어 페이지 (개별 에피소드를 보는 페이지)

### ③ [웹툰/만화의 경우] 본문 이미지 컨테이너
- **필요한 것:** 웹툰 본문 이미지들이 나열되어 있는 메인 컨테이너 영역과 그 안의 **`<img>` 태그 1~2개**의 HTML 구조
- 💡 **중요 포인트:** 일부 사이트는 로딩 속도 최적화를 위해 원본 이미지 주소를 `src`가 아닌 `data-src`, `data-lazy` 같은 커스텀 속성에 숨겨놓는 경우가 많으므로, 이 속성들이 잘 보이도록 복사해야 합니다.
- **추출 목적:** 다운로드할 원본 이미지 URL 목록을 추출하기 위함 (`getImageList` 구현)

### ④ [소설의 경우] 텍스트 본문 영역 (해당 시)
- **필요한 것:** 소설의 텍스트가 들어있는 메인 본문 컨테이너(예: `<div id="novel_content">...</div>`)의 HTML 구조
- **추출 목적:** 소설 텍스트 데이터를 통째로 추출하기 위함 (`getNovelContent` 구현)

---

## 3. 추가 필수 정보
- **사이트 URL 패턴:** 대상 사이트의 기본 도메인 주소와 뷰어/목록 페이지의 URL 형태 예시 
  - *예시: `https://example.com/webtoon/12345`*

---

## 4. 실제 추출 예시 (Case Study: 블랙툰)

`PARSER_DOM_WEBTOON.md` 리포트를 바탕으로 한 실제 DOM 추출 및 분석 요약본입니다. 새로운 파서를 제작할 때 어떤 데이터를 찾아야 하는지 참고하세요.

- **메타 데이터 (`div.hero-v2-meta`, `div.hero-v2-thumb`)**:
  - 작품 제목: `h1.hero-v2-title`
  - 썸네일: `div.hero-v2-thumb > img`
  - 작가: `div.hero-v2-author`
- **에피소드 목록 (`ul.ep-list-v2`)**:
  - 개별 항목: `li.ep-row-v2`
  - 화 번호: `span.ep-row-v2-no`
  - 회차 제목: `.ep-row-v2-title strong`
  - 링크: `a.ep-row-v2-link`

*(이 추출 데이터를 기반으로 완성된 JSON 규칙은 `DYNAMIC_RULE_GUIDE.md`의 예시를 참고하세요.)*
