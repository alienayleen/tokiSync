# 📥 다운로드 안정성 및 순서 최적화 보고서 (v2.1)

본 문서는 다운로드 속도 정책 확장, API 복호화 시 안정성 강화, 그리고 범위 다운로드 순서 정정 작업에 대한 기획 및 수행 내역을 기록합니다.

---

## 1. [PLANNING PHASE] 기획 및 분석

### [분석]
- **속도 정책 제한**: 기존 3단계(최대 8초) 정책으로는 서버의 공격 방어 기제(WAF, DDoS 감지)를 회피하기에 부족함이 있었음.
- **API 복호화 위험**: 소설 복호화(`novel-decryptor.js`) 시 대기 시간 없이 연속 호출될 경우 차단 위험이 매우 높음.
- **다운로드 순서 불일치**: 사이트의 역순(최신순) 정렬 특성상, 범위 지정 시 마지막 회차부터 다운로드되는 사용자 경험 저하 발생.

### [기획 목표]
1. 다운로드 속도 단계를 5단계로 확장하고 최대 30초 대기 옵션 제공.
2. API 방식 호출 시 사용자의 설정과 관계없이 가장 안전한 '매우 느림' 단계 강제 적용.
3. 범위 다운로드 시 에피소드 번호 기준 오름차순(1화 -> 10화) 정렬 보장.

---

## 2. [IMPLEMENTATION] 수행 내역

### 🛠️ 수정된 파일 및 로직

#### 1) `src/core/downloader.js`
- **SLEEP_POLICIES 확장**: `slow` (5-15초), `very_slow` (10-30초) 정책 추가.
- **API 안정성 강화**: `processItem` 함수에서 `fetchMethod === 'api'`일 경우 `policy`를 `very_slow`로 강제 고정하고, `fetchNovelText` 호출 직후 `await sleep()`을 수행하도록 변경.
- **정렬 로직 수정**: `tokiDownload` 함수에서 `rangeSet`이 존재할 경우, 목록 아이템을 파싱하여 번호(`num`) 기준 오름차순으로 정렬한 뒤 처리 루프에 전달.

#### 2) `src/core/ui.js`
- **설정 UI 반영**: 메뉴 모달(`toki-sel-speed`)에 '느림 (5-15초)', '매우 느림 (10-30초)' 옵션 태그 추가.

---

## 3. [SELF-AUDIT] 자가 검증 및 결과

- **논리 검증**: 
    - API 모드에서 사용자가 '빠름'을 선택했더라도 코드 수준에서 `very_slow`로 덮어쓰므로 보안 정책이 우선됨을 확인.
    - 정렬 시 `parser.parseListItem`을 반복 호출하지 않도록 `map`을 활용해 성능 및 데이터 정합성 유지.
- **예상 결과**: 
    - 소설 복호화 다운로드 시 서버 부하를 최소화하여 IP 차단 확률 감소.
    - 부분 다운로드 시 사용자가 의도한 진행 방향(순차적 저장)으로 동작.
