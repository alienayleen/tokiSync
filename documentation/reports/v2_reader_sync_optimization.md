# 리포트: V2 소설 뷰어 툴바 이식 및 진도 동기화 정밀화 (v2.9.2)

이 리포트는 V2 소설 뷰어의 사용자 경험(UX)을 개선하고, 폰트 및 레이아웃 변경 시 발생하던 페이지 이탈 현상을 근본적으로 해결한 과정과 기술적 상세 내용을 담고 있습니다.

## 1. 개요
*   **배경**: V2 뷰어는 현대적인 아키텍처를 가졌으나, V1에서 제공하던 텍스트 설정(테마, 폰트 크기 등) 기능이 누락되어 있었고, 페이지 모드에서 폰트 크기를 바꾸면 읽던 위치를 잃어버리는 고질적인 문제가 있었습니다.
*   **목표**:
    1. V1의 텍스트 설정 기능을 V2 플로팅 푸터에 성공적으로 이식.
    2. 소설 가독성을 위한 가로폭(720px) 표준화.
    3. 어떠한 레이아웃 변화에도 읽던 문단을 정확히 찾아가는 **DOM Offset 기반 Locator 시스템** 구축.

## 2. 주요 해결 과제 및 기술 전략

### 2.1. 텍스트 설정 툴바 이식
*   **UI/UX**: 뷰어 하단 영역에 테마(Sepia, Dark 등), 폰트 크기, 줄 간격 조절 인터페이스를 배치했습니다.
*   **반응성**: 사용자가 설정을 변경하면 `TextRenderer.vue`에 즉시 반영되며, 동시에 페이지 재계산(@paginate)을 트리거하여 정합성을 유지합니다.

### 2.2. DOM Offset 기반 정밀 트래킹 (Locator System)
기존의 비율 기반(Heuristic) 계산은 폰트가 조금만 커져도 수십 페이지의 오차가 발생했습니다. 이를 해결하기 위해 **문단(Paragraph) 기반 추적** 방식을 도입했습니다.
*   **저장 로직**: 현재 화면(Viewport)의 맨 앞에 위치한 문단의 `data-locator`(인덱스)를 찾아 `logicalIndex`로 저장합니다.
*   **복원 로직**: 페이지 모드에서 특정 인덱스를 찾을 때, 해당 DOM 요소의 `offsetLeft` 값을 뷰어의 `clientWidth`로 나누어 정확히 몇 번째 페이지(Column)에 있는지 1:1로 계산합니다.
*   **효과**: 폰트 크기를 극단적으로 키우거나 줄여도, 다시 계산된 페이지 번호로 정확하게 이동합니다.

### 2.3. 에피소드 전환 시 데이터 무결성 확보 (Flush & Reset)
새 작품을 열 때 이전 작품의 진도가 잔류하여 페이지가 건너뛰어지는 현상을 방지하기 위해 이중 안전장치를 마련했습니다.
*   **즉시 저장 (Flush)**: 에피소드 이동 직전, 현재 위치를 DB에 즉시 저장하여 데이터 유실을 방지합니다.
*   **메모리 초기화 (Reset)**: 새 에피소드 로드 시 `logicalIndex`를 0으로 강제 초기화하여 오염을 원천 차단합니다.

## 3. 기술 사양 (v2.9.2)

### 핵심 코드 구조
*   **`useProgressMarker.js`**: `flushSaveToDB`, `resetLocator`, `getPageFromLocator` (DOM 기반 계산 엔진).
*   **`useStore.js`**: `startReading` 시 Flush 및 Reset 호출 순서 보장.
*   **`TextRenderer.vue`**: 가로폭 `720px` 고정 및 `column-width` 자동 계산 로직.

## 4. 최종 결과 요약
*   ✅ **독서 환경 최적화**: 720px 가로폭과 테마 지원으로 전문 전자책 수준의 가독성 확보.
*   ✅ **진도 동기화 완결**: 설정 변경, 창 크기 조절, 에피소드 이동 시에도 "읽던 곳"을 1px의 오차 없이 보존.
*   ✅ **안정성**: 비동기 저장 지연으로 인한 데이터 롤백 버그 해결.
